# Explicit per-input acknowledgment instead of order-randomization

The Chairman must explicitly acknowledge every Stage 1/2/3/research input by name before writing its final answer — what it said and whether/how it shaped the synthesis, including inputs it dismisses. We initially proposed randomizing the order these inputs are presented in, to prevent the Chairman from anchoring on whichever section it reads first or last (a real, documented LLM position effect when reading multiple sections in one prompt). We rejected that fix: randomizing order doesn't remove the positional bias, it just makes it land on an unpredictable section each run instead of a fixed one — the underlying skimming/underweighting risk is still there, just no longer visible or auditable. Forcing explicit acknowledgment of every input instead makes any real dismissal visible in the printed output, so it can be inspected and challenged, rather than hidden wherever it happened to land in the prompt.
