# Abort the council run on any agent failure, don't degrade gracefully

If any Stage 1/2/3 Agent call fails, `/council` stops the run entirely rather than continuing with a noted gap — a deliberate departure from upstream `llm-council`'s graceful-degradation philosophy ("continue with successful responses if some models fail"). The reasoning: every agent currently runs on the same Claude subscription, so a single failure is more likely a systemic issue (rate limit, usage cap, outage) than an isolated model-specific fluke, and continuing past it risks feeding the Chairman a silent gap instead of surfacing a problem worth investigating first. Once local models are added (see `CLAUDE.md`), a single model's failure becomes a more plausible isolated event, and this decision should be revisited.
